# SerialRUN 使用指南

SerialRUN 是一款功能丰富的串口调试助手，支持串口通信、Modbus 调试、PLC 控制、CAN 总线分析、I2C/SPI 调试、固件烧录等多种功能。内置 MCP 服务器，允许 AI 助手通过 TCP 远程控制串口设备。

---

## 快速开始

1. 将串口设备通过 USB 连接到电脑
2. 点击左侧面板的刷新按钮 `↻` 检测端口
3. 从下拉列表中选择串口设备（如 COM3）
4. 设置波特率（默认 115200，大多数设备适用）
5. 点击绿色「连接」按钮
6. 在底部终端输入框输入命令，点击「发送」

如果不确定波特率，可以点击 **Auto** 按钮自动检测。

- 绿色 `↓ RX` — 接收的数据

- 黄色 `⚙ SYS` — 系统消息

- **发送HEX** — 切换十六进制输入模式
- **接收HEX** — 切换接收数据显示模式
- **显示时间戳** — 每条消息前显示时间（格式：年-月-日 时:分:秒.毫秒）
- **自动滚动** — 自动滚动到最新消息
- **CRC** — 发送时自动追加校验码（8 种算法）

## 功能详解

### 日志查看器

记录所有系统事件，包括连接/断开、数据收发、错误信息等。
- 支持 INFO/WARN/ERR 三级日志，颜色区分
- 显示完整的收发数据（HEX + 文本）
- 自动持久化到 `~/.serialrun/logs.json`，重启不丢失
- **清空** — 清除所有日志
- **导出** — 导出为 CSV 文件（包含时间戳、级别、消息）

### 数据图表

实时显示数据收发速率（字节/秒），Y 轴自动缩放，显示网格线。
- 底部显示累计 RX/TX 字节计数
- 最大值标签显示当前最高速率

### Modbus 调试工具

三个可折叠区域：

**快速请求**
- 从站地址：0-247
- 功能码：01 读线圈 / 02 读离散输入 / 03 读保持寄存器 / 04 读输入寄存器 / 05 写单个线圈 / 06 写单个寄存器 / 15 写多个线圈 / 16 写多个寄存器
- 起始地址和数量/写入值
- 显示最近一次请求和响应的十六进制数据

**寄存器监控**
- 设置轮询间隔（100-10000ms）
- 实时显示各寄存器的地址、原始值、格式化值和最后更新时间

**帧日志**
- 记录所有 Modbus 帧的收发历史（最多 200 条）
- 显示时间戳、请求/响应十六进制和解码内容

### PLC 控制器

专业极简的 PLC 监控面板：
- **品牌**：西门子 S7-1200、三菱 FX3U、台达 DVP、欧姆龙 CP1H、自定义
- **数据类型**：BOOL、UINT16、INT16、UINT32、FLOAT32
- **批量读取**：连续寄存器自动合并为单次 Modbus 请求，轮询更快
- **内联写入**：点击寄存器行即可编辑和写入值
- **缩放系数**：读取时自动缩放，写入时反向缩放
- **逐寄存器状态**：绿/黄/红点指示数据新鲜度（<3s / <10s / 过期）
- **自动轮询**：可配置间隔（100-10000ms）

### TCP/RTU 桥接

将 Modbus TCP 客户端桥接到串口 RTU 设备。外部 SCADA/HMI 软件可通过 TCP 访问串口设备。

- **TCP 端口** — TCP 监听端口（默认 502）
- **串口** — 连接 RTU 设备的串口
- **波特率** — 串口通信速率
- **超时** — 响应超时时间（毫秒）
- **启动/停止** — 切换桥接运行状态
- **桥接日志** — 显示所有桥接的请求/响应及时间戳

### HMI 模拟器

模拟虚拟 Modbus 从站设备。用于测试 PLC 程序、SCADA 系统或 Modbus 主站软件，无需物理硬件。

- **模式** — TCP 服务器（监听 TCP 连接）或 RTU 从站（在串口上响应）
- **TCP 端口** — 监听端口（TCP 模式）
- **串口/波特率** — 串口设置（RTU 模式）
- **从站地址** — Modbus 从站地址（0-247）
- **保持寄存器** — 编辑寄存器值（0-65535），添加新寄存器
- **线圈** — 切换线圈开关状态，添加新线圈
- **模拟器日志** — 显示所有收到的请求和响应

### 文件传输

通过串口传输文件，支持以下协议：
- **XMODEM** — 基础校验
- **XMODEM-CRC** — CRC 校验，更可靠
- **YMODEM** — 支持文件名和大小信息
- **ZMODEM** — 高级协议，支持断点续传

### 帧生成器

手动构造 Modbus 帧：
1. 设置从站 ID、功能码、地址、值
2. 点击「构建」生成十六进制帧
3. 点击「发送」通过串口发送

### 数据记录

将串口收发数据记录到 CSV 文件：
- 记录时间戳、方向（TX/RX）、数据（文本和十六进制）、字节数
- 显示当前缓冲区中的记录数
- 可随时开始/停止记录

### CAN 总线分析器

基于 SLCAN 协议的 CAN 总线分析工具：
- **采集** — 开始/停止捕获 CAN 帧
- **统计** — 总帧数、错误帧数、唯一 ID 数、总线负载估算
- **ID 统计** — 按 ID 显示帧数、频率、时间间隔
- **帧列表** — 显示时间戳、方向、ID（标准/扩展）、DLC、数据
- **发送** — 输入 ID 和数据发送 CAN 帧
- **过滤** — 按 ID 过滤显示
- 支持 CSV 导出，最多保存 2000 帧

### I2C/SPI 调试

两种模式切换：

**I2C 模式**
- 设置设备地址（默认 0x68）和寄存器地址
- **扫描** — 扫描 0x08-0x77 范围内的 I2C 设备
- **读取** — 从指定寄存器读取数据
- **写入** — 向指定寄存器写入数据

**SPI 模式**
- 输入 MOSI 数据（十六进制）
- **传输** — 发送数据并接收响应

### 示波器

将串口数据可视化为波形图：
- 可配置时间基准（1-5000ms）
- 10x16 网格，Y 轴自动缩放
- 鼠标悬停显示精确时间和数值
- 最大缓冲 10000 个数据点

### 固件烧录器

支持两种 MCU 的固件烧录：

**STM32**
- 连接时发送 0x7F 初始化序列
- 支持 HEX 和 BIN 格式固件
- 分块写入（128 字节/块）

**ESP32**
- 连接时发送同步序列
- 支持 BIN 和 ELF 格式固件
- 分块写入（128 字节/块）

### 寄存器编辑器

自定义设备寄存器映射表：
- 添加/删除寄存器条目
- 设置地址、名称、数据类型、值、描述
- 支持从 CSV/JSON 导入，导出为 CSV
- 可选报警功能，设置阈值

### 插件管理器

扫描 `plugins/` 目录下的扩展插件：
- 显示插件名称、版本、作者和加载状态
- 支持 Windows (.dll)、Linux (.so)、macOS (.dylib) 平台
- 点击刷新按钮重新扫描

## 使用技巧

- 调试新设备时，建议先从 **115200** 波特率开始
- 如果不知道波特率，使用 **Auto** 按钮自动检测
- HEX 模式下发送数据时，会自动过滤空格和 `0x` 前缀
- 发送文本时可选择追加 **CRC 校验码**，确保数据完整性
- **自动回复** 功能适合需要应答的协议调试
- 使用 **录制/回放** 功能可快速重复测试序列
- Modbus 调试时，先用「快速请求」验证连通性，再用「寄存器监控」持续观察
- PLC 控制器支持自定义品牌，可导入自己的寄存器定义
- 所有配置和日志自动保存到 `~/.serialrun/` 目录，重启不丢失

## 录制与回放

SerialRUN 支持录制串口操作并回放，用于重复测试序列。

### 录制操作

1. 在左侧面板找到「录制 / 回放」区域
2. 点击 **开始录制** — 状态栏显示红色 `● 录制中`
3. 正常使用终端发送数据（所有发送的命令都会被记录，包括命令间隔）
4. 点击 **停止录制** — 显示已录制的命令数量

### 保存脚本

录制完成后，点击 **保存** 按钮：
- 支持 `.txt` 和 `.srs` 格式
- 脚本文件为纯文本，每行一条命令：
  - `SEND 数据内容` — 发送命令
  - `WAIT 毫秒数` — 等待指定时间

### 加载脚本

点击 **导入** 按钮加载之前保存的脚本文件。

### 回放操作

1. 确保已连接串口设备
2. 点击绿色 **▶ 回放** 按钮
3. 脚本按录制的延时顺序自动执行
4. 进度条显示当前进度
5. 随时可点击红色 **■ 停止** 中断回放

### 脚本文件示例

```
# SerialRUN Script
SEND AT
WAIT 500
SEND AT+RST
WAIT 1000
SEND AT+CWMODE=1
```

## 终端高级功能

### HEX 模式

终端支持独立的发送/接收 HEX 模式：
- **发送HEX** — 勾选后，输入框中的内容按十六进制解析发送（如 `48 65 6C 6C 6F`）
- **接收HEX** — 勾选后，接收到的数据以空格分隔的十六进制显示（如 `4F 4B 0D 0A`）
- 支持 `0x` 前缀（如 `0x48 0x65`），自动过滤空格

### 行尾符

发送文本时可自动追加行尾符：
- **无** — 不追加任何字符
- **CR (\r)** — 回车符，常用于串口终端
- **LF (\n)** — 换行符，常用于网络协议
- **CRLF (\r\n)** — 回车+换行，HTTP/Modbus ASCII 标准

### 自动发送

点击 **自动发送** 按钮后，输入框中的内容将按设定间隔自动重复发送：
- 可配置间隔时间（100ms - 60s）
- 适用于心跳包、轮询请求等场景
- 再次点击 **停止发送** 可取消

### DTR/RTS 控制

连接状态下，终端工具栏显示 DTR 和 RTS 复选框：
- **DTR (Data Terminal Ready)** — 数据终端就绪信号
- **RTS (Request To Send)** — 请求发送信号
- 某些设备（如 Arduino）需要 DTR 信号才能正常通信

### 保留输入

终端输入框左侧的 **保留输入** 复选框：
- 勾选后，发送数据不会清空输入框
- 适用于需要反复发送相同命令的调试场景
- 默认关闭（发送后自动清空）

### 校验码

发送时可自动追加校验码，支持 8 种算法：
- **CRC-16/MODBUS** — Modbus RTU 标准校验（多项式 0xA001）
- **CRC-16/CCITT** — CCITT 标准（多项式 0x1021）
- **CRC-16/XMODEM** — XMODEM 文件传输协议校验
- **CRC-32** — 32位循环冗余校验（ZIP/以太网）
- **LRC** — 纵向冗余校验（Modbus ASCII）
- **Checksum-8** — 8位累加和
- **Checksum-16** — 16位累加和

鼠标悬停在校验码面板中的算法名称上可查看详细说明。

## MCP 服务器

SerialRUN 内置 MCP (Model Context Protocol) 服务器，允许 AI 助手通过 TCP 远程控制串口设备。
所有串口操作通过 GUI 端口管理器路由，确保与 GUI 操作互不冲突。

### 启用与配置

在左侧设置面板中找到「MCP 服务器」：
1. 勾选「启用 MCP 服务器」
2. 设置端口号（默认 9527）
3. 选择绑定地址：
   - **仅本机** — 只允许本机 AI 助手连接（推荐）
   - **所有接口（局域网）** — 允许局域网内的 AI 助手连接

### 可用工具（11 个）

| 工具 | 说明 |
|------|------|
| `list_ports` | 列出所有可用串口设备 |
| `connect` | 连接串口（支持设置波特率、数据位、停止位、校验位） |
| `disconnect` | 断开当前连接 |
| `send` | 发送数据（支持文本和十六进制） |
| `read` | 读取数据（带超时） |
| `send_command` | 发送命令并等待响应（写入-读取模式） |
| `modbus_read` | 读取 Modbus RTU 保持寄存器 |
| `modbus_write` | 写入 Modbus RTU 保持寄存器 |
| `plc_read` | 读取 PLC 预设品牌的所有寄存器 |
| `plc_write` | 按地址写入 PLC 寄存器 |
| `get_access_log` | 查看访问日志（客户端 IP、操作记录、时间戳） |

### 使用方法

1. 打开帮助面板（点击 `?` 按钮）
2. 在 MCP 服务器部分，点击 **复制 MCP 说明** 按钮
3. 将复制的内容粘贴给任何支持 MCP 的 AI 助手
4. AI 助手即可通过 TCP 控制你的串口设备

AI 助手可使用 `tools/list` 方法获取所有可用工具及其参数说明。

### 访问日志

- 所有 MCP 操作自动记录客户端 IP 地址
- 左侧设置面板显示最近 20 条访问记录
- 颜色编码：🟢 CONNECT、🔴 DISCONNECT、🔵 CALL
- 可通过 `get_access_log` 工具查询完整日志

### 局域网模式

启用局域网模式后：
- 显示本机 IP 地址和端口（如 `192.168.1.100:9527`）
- 局域网内其他设备的 AI 助手可连接控制
- 所有操作自动记录客户端 IP，可通过 `get_access_log` 工具查看
- ⚠️ 请谨慎使用，网络中任何人都可以控制串口

|------|------|----------|
| `config.toml` | 主题、语言、波特率等配置 | 切换设置时 |

## 常见问题

**Q: 端口列表为空？**
A: 点击刷新按钮 `↻`。如果仍然为空，检查设备是否已连接、驱动是否安装。Linux 用户可能需要 
`sudo usermod -a -G dialout $USER` 添加串口权限。

**Q: 连接后没有收到数据？**
A: 检查波特率是否匹配设备设置，确认接线正确（TX/RX 交叉连接），检查流控设置。

**Q: 自动波特率检测失败？**
A: 确保设备正在发送数据。自动检测需要设备在连接期间有数据输出。

**Q: Modbus 通信无响应？**
A: 检查从站地址是否正确，确认功能码和地址范围，验证 CRC 校验。

**Q: PLC 寄存器显示 ERR？**
A: 寄存器读取失败。检查接线、从站地址，确认该寄存器地址在 PLC 上存在。

**Q: 如何使用 MCP 服务器？**
A: MCP 服务器默认启用，监听端口 9527（仅本机）。在设置 > MCP 服务器中配置：
- 修改端口号（如果 9527 被占用）
- 启用「所有接口（局域网）」允许远程 AI 助手连接
- AI 助手可通过 MCP 协议控制串口设备（详见帮助 > MCP 说明）

**Q: 文件传输失败？**
A: 确保两端使用相同的协议，检查串口连接稳定，尝试降低波特率。

**Q: 重启后数据丢失？**
A: 所有数据自动保存到 `~/.serialrun/` 目录。检查该目录是否存在，是否有写入权限。
