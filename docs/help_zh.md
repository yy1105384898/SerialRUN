# SerialTap 使用指南

SerialTap 是一款功能丰富的串口调试助手，支持串口通信、Modbus 调试、PLC 控制、CAN 总线分析、I2C/SPI 调试、固件烧录等多种功能。内置 MCP 服务器，允许 AI 助手通过 TCP 远程控制串口设备。

---

## 快速开始

1. 将串口设备通过 USB 连接到电脑
2. 点击左侧面板的刷新按钮 `↻` 检测端口
3. 从下拉列表中选择串口设备（如 COM3）
4. 设置波特率（默认 115200，大多数设备适用）
5. 点击绿色「连接」按钮
6. 在底部终端输入框输入命令，按回车或点击「发送」

如果不确定波特率，可以点击 **Auto** 按钮自动检测。

## 界面布局

### 顶部工具栏

工具栏包含 13 个功能按钮，点击可打开/关闭对应的功能窗口：

| 按钮 | 功能 | 说明 |
|------|------|------|
| Log | 日志查看器 | 显示收发数据记录，支持导出 CSV |
| Chart | 数据图表 | 实时绘制数据速率曲线 |
| PLC | PLC 控制 | 支持西门子、三菱、台达、欧姆龙等品牌 |
| Mod | Modbus 调试 | 快速读写寄存器，支持 8 种功能码 |
| FT | 文件传输 | 支持 XMODEM/YMODEM/ZMODEM 协议 |
| FB | 帧生成器 | 手动构造 Modbus 帧并发送 |
| DL | 数据记录 | 将串口数据记录到 CSV 文件 |
| CAN | CAN 总线 | 捕获和解析 CAN 帧（SLCAN 协议） |
| I2C | I2C/SPI | 扫描和读写 I2C/SPI 设备 |
| Scope | 示波器 | 可视化串口数据波形 |
| Flash | 烧录器 | 支持 STM32/ESP32 固件烧录 |
| Reg | 寄存器编辑 | 自定义设备寄存器映射表 |
| Plug | 插件管理 | 加载和管理扩展插件 |

工具栏右侧有三个系统按钮：
- **?** — 打开本使用指南
- **Dark/Light** — 切换深色/浅色主题
- **EN/中** — 切换英文/中文界面

### 左侧面板

- **串口选择** — 从下拉列表选择串口设备，点击 `↻` 刷新端口列表
- **波特率** — 选择通信速率：9600、19200、38400、57600、115200、230400、460800、921600
- **Auto 按钮** — 自动检测波特率，依次尝试各速率直到收到数据
- **串口配置** — 数据位（5-8）、停止位（1/2）、校验位（无/奇/偶）、流控（无/软件/硬件）
- **显示设置** — 十六进制模式、时间戳显示、自动滚动
- **自动回复** — 设置匹配模式和回复内容，收到匹配数据时自动发送回复
- **录制/回放** — 录制串口操作脚本，保存后可回放

### 终端区域

终端显示所有收发数据，采用颜色区分方向：
- 绿色 `↓ RX` — 接收的数据
- 蓝色 `↑ TX` — 发送的数据
- 黄色 `⚙ SYS` — 系统消息

顶部工具栏可设置：
- **HEX** — 切换十六进制输入模式
- **显示时间戳** — 每条消息前显示时间
- **自动滚动** — 自动滚动到最新消息
- **CRC** — 发送时自动追加校验码（CRC16/MODBUS、CRC16/CCITT、CRC32、LRC、SUM8）

## 功能详解

### 日志查看器

记录所有系统事件，包括连接/断开、数据收发、错误信息等。
- 支持 INFO/WARN/ERR 三级日志，颜色区分
- 显示当前日志条目数量
- **清空** — 清除所有日志
- **导出** — 导出为 CSV 文件（包含时间戳、级别、消息）

### 数据图表

实时显示数据收发速率（字节/秒），Y 轴自动缩放，显示网格线。
- 底部显示累计 RX/TX 字节计数
- 最大值标签显示当前最高速率

### Modbus 调试工具

三个可折叠区域：

**快速请求**
- 从站地址：0-247
- 功能码：01 读线圈 / 02 读离散输入 / 03 读保持寄存器 / 04 读输入寄存器 / 05 写单个线圈 / 06 写单个寄存器 / 15 写多个线圈 / 16 写多个寄存器
- 起始地址和数量/写入值
- 显示最近一次请求和响应的十六进制数据

**寄存器监控**
- 设置轮询间隔（100-10000ms）
- 实时显示各寄存器的地址、原始值、格式化值和最后更新时间

**帧日志**
- 记录所有 Modbus 帧的收发历史（最多 200 条）
- 显示时间戳、请求/响应十六进制和解码内容

### PLC 控制器

支持主流 PLC 品牌的 Modbus 通信：
- **品牌**：西门子、三菱、台达、欧姆龙、自定义
- **数据类型**：BOOL、UINT16、INT16、UINT32、FLOAT
- 寄存器表显示地址、名称、类型、值、缩放系数、单位
- BOOL 寄存器可通过复选框切换开关状态
- 支持全部读取和单次读取
- 自动轮询模式，可配置间隔（200-10000ms）

### 文件传输

通过串口传输文件，支持以下协议：
- **XMODEM** — 基础校验
- **XMODEM-CRC** — CRC 校验，更可靠
- **YMODEM** — 支持文件名和大小信息
- **ZMODEM** — 高级协议，支持断点续传

操作流程：选择协议 → 点击发送/接收 → 选择文件 → 等待传输完成

### 帧生成器

手动构造 Modbus 帧：
1. 设置从站 ID、功能码、地址、值
2. 点击「构建」生成十六进制帧
3. 点击「发送」通过串口发送

### 数据记录

将串口收发数据记录到 CSV 文件：
- 记录时间戳、方向（TX/RX）、数据（文本和十六进制）、字节数
- 显示当前缓冲区中的记录数
- 可随时开始/停止记录

### CAN 总线分析器

基于 SLCAN 协议的 CAN 总线分析工具：
- **采集** — 开始/停止捕获 CAN 帧
- **统计** — 总帧数、错误帧数、唯一 ID 数、总线负载估算
- **ID 统计** — 按 ID 显示帧数、频率、时间间隔
- **帧列表** — 显示时间戳、方向、ID（标准/扩展）、DLC、数据
- **发送** — 输入 ID 和数据发送 CAN 帧
- **过滤** — 按 ID 过滤显示
- 支持 CSV 导出，最多保存 2000 帧

### I2C/SPI 调试

两种模式切换：

**I2C 模式**
- 设置设备地址（默认 0x68）和寄存器地址
- **扫描** — 扫描 0x08-0x77 范围内的 I2C 设备
- **读取** — 从指定寄存器读取数据
- **写入** — 向指定寄存器写入数据

**SPI 模式**
- 输入 MOSI 数据（十六进制）
- **传输** — 发送数据并接收响应

### 示波器

将串口数据可视化为波形图：
- 可配置时间基准（1-5000ms）
- 10x16 网格，Y 轴自动缩放
- 鼠标悬停显示精确时间和数值
- 最大缓冲 10000 个数据点

### 固件烧录器

支持两种 MCU 的固件烧录：

**STM32**
- 连接时发送 0x7F 初始化序列
- 支持 HEX 和 BIN 格式固件
- 分块写入（128 字节/块）

**ESP32**
- 连接时发送同步序列
- 支持 BIN 和 ELF 格式固件
- 分块写入（128 字节/块）

操作：选择 MCU 类型 → 连接 → 选择固件文件 → 擦除（可选）→ 烧录

### 寄存器编辑器

自定义设备寄存器映射表：
- 添加/删除寄存器条目
- 设置地址、名称、数据类型、值、描述
- 支持从 CSV/JSON 导入，导出为 CSV
- 可选报警功能，设置阈值

### 插件管理器

扫描 `plugins/` 目录下的扩展插件：
- 显示插件名称、版本、作者和加载状态
- 支持 Windows (.dll)、Linux (.so)、macOS (.dylib) 平台
- 点击刷新按钮重新扫描

## 使用技巧

- 调试新设备时，建议先从 **115200** 波特率开始
- 如果不知道波特率，使用 **Auto** 按钮自动检测
- HEX 模式下发送数据时，会自动过滤空格和 `0x` 前缀
- 发送文本时可选择追加 **CRC 校验码**，确保数据完整性
- **自动回复** 功能适合需要应答的协议调试
- 使用 **录制/回放** 功能可快速重复测试序列
- Modbus 调试时，先用「快速请求」验证连通性，再用「寄存器监控」持续观察
- PLC 控制器支持自定义品牌，可导入自己的寄存器定义

## 常见问题

**Q: 端口列表为空？**
A: 点击刷新按钮 `↻`。如果仍然为空，检查设备是否已连接、驱动是否安装。Linux 用户可能需要 `sudo usermod -a -G dialout $USER` 添加串口权限。

**Q: 连接后没有收到数据？**
A: 检查波特率是否匹配设备设置，确认接线正确（TX/RX 交叉连接），检查流控设置。

**Q: 自动波特率检测失败？**
A: 确保设备正在发送数据。自动检测需要设备在连接期间有数据输出。

**Q: Modbus 通信无响应？**
A: 检查从站地址是否正确，确认功能码和地址范围，验证 CRC 校验。

**Q: 文件传输失败？**
A: 确保两端使用相同的协议，检查串口连接稳定，尝试降低波特率。
